# Отчёт о качестве данных

## 1. Введение

**Датасет:** Нобелевские лауреаты (1901-2016)  
**Источник:** Открытые данные Nobel Prize API / сборный датасет  
**Размер:** 911 строк × 23 столбца  

**Предметная область:** История науки и наукометрия. Данные содержат информацию о лауреатах Нобелевской премии: год присуждения, категория, имя, дата рождения, страна рождения, пол, возраст, организация, дата смерти и другие демографические характеристики.

**Условный заказчик:** Нобелевский фонд или исследовательская группа, изучающая факторы, влияющие на присуждение премий, а также социологи и наукометристы.

**Возможные задачи ИАД:**
1. Сегментация лауреатов по возрасту, полу, стране и категории премии
2. Поиск аномалий (экстремально молодые или возрастные лауреаты)
3. Описательная аналитика (динамика премий по годам, распределение по странам)

## 3. Основные проблемы качества данных

| № | Тип проблемы | Описание | Потенциальный риск | Предлагаемое решение |
|---|--------------|----------|-------------------|---------------------|
| 1 | Некорректные значения | В столбце `age` присутствуют значения 0.0 (невозможный возраст) | Искажение статистики среднего возраста | Удалить строки с возрастом 0 или заменить на NaN |
| 2 | Пропуски | В столбцах об организации (`organization_name`, `organization_city`, `organization_country`) пропущено более 50% данных | Ограниченный анализ по организациям | При анализе исключать пропуски или выделить их в отдельную категорию |
| 3 | Пропуски | В столбцах `death_date`, `death_city`, `death_country` ~30% пропусков | Некорректный расчёт продолжительности жизни | Учитывать только записи с полными данными для анализа продолжительности жизни |
| 4 | Грязные категории | В столбце `birth_country` встречаются исторические названия со скобками, например "Prussia (Poland)" | Затруднён анализ по современным странам | Создать новый столбец с извлечением современного названия страны |
| 5 | Пропуски | В столбце `birth_date` есть единичные пропуски (2 записи) | Незначительное влияние | Можно игнорировать или удалить строки |
| 6 | Дубликаты | Полных дубликатов строк не найдено | - | - |
| 7 | Выбросы | В столбце `age` есть экстремальные значения (17 лет, 90+ лет) | Не ошибка, но требует отдельного рассмотрения | Не удалять, но учитывать при интерпретации результатов |

## 4. План улучшения качества

**Приоритет 1 (критично):**
- Исправление возраста: удалить строки с возрастом 0
- Обработка пропусков в данных об организациях (исключить или выделить в отдельную категорию)

**Приоритет 2:**
- Создание нового признака — современное название страны из столбца `birth_country`
- Для расчёта продолжительности жизни использовать только записи с полными данными (`death_date` не пусто)

**Приоритет 3:**
- Анализ выбросов возраста как отдельные кейсы
- Удалить строки с пропусками в `birth_date` (2 записи)

## 5. Краткое резюме

В ходе работы были выявлены следующие ключевые проблемы: присутствуют пропуски в важных для анализа полях (организации — более 50%, даты смерти — около 30%), обнаружены некорректные значения (возраст 0), категориальные признаки содержат "грязные" значения (исторические названия стран в поле `birth_country`). При дальнейшем анализе особого внимания требуют корректная обработка пропусков, учёт выбросов (молодые и возрастные лауреаты) как особых случаев и очистка географических данных. Основные выводы по распределению премий по категориям и возрасту лауреатов можно считать надёжными, так как ключевые столбцы (`year`, `category`, `age`) содержат преимущественно корректные данные.
